**FINAL PROJECT — EVENT SYSTEM (EVENT MANAGEMENT & REGISTRATION)** 

**1\. TUJUAN PROJECT** 

Kalian WAJIB membangun sebuah sistem **Event Management & Registration System** berbasis **Mobile \+ Web \+ Backend** yang mampu: 

• mengelola pendaftaran peserta event,  

• mengelola berbagai jenis event (seminar, konser, gathering, dll),  • mengatur tempat duduk atau slot peserta,  

• melakukan check-in dan check-out menggunakan QR Code,  

• menyediakan fitur lucky draw (undian),  

• menghasilkan sertifikat digital dengan QR verifikasi,  

• menyediakan dashboard monitoring untuk admin.  

Sistem ini HARUS mampu menangani event dengan kompleksitas berbeda, seperti: • event sederhana (seminar tanpa seat mapping),  

• event kompleks (gathering \+ hotel \+ ballroom \+ seat layout).  

Project ini bukan sekadar form pendaftaran event. 

Project ini adalah simulasi **sistem event management profesional**. 

**2\. MASALAH YANG HARUS KALIAN SELESAIKAN** 

Dalam pengelolaan event manual: 

• pendaftaran tidak terstruktur,  

• sulit mengatur kapasitas peserta,  

• tidak ada sistem tempat duduk,  

• check-in manual dan rawan kesalahan,  

• tidak ada tracking kehadiran,   
• tidak ada sistem undian digital,  

• tidak ada sertifikat otomatis,  

• tidak ada validasi keaslian sertifikat.  

 Kalian HARUS menyelesaikan masalah tersebut dengan membangun sistem yang: • mengelola registrasi secara digital,  

• mengatur kapasitas dan slot peserta,  

• mengatur tempat duduk atau kamar (jika ada),  

• menyediakan QR check-in,  

• menyediakan lucky draw otomatis,  

• menghasilkan sertifikat digital yang bisa diverifikasi.

**7.3 Category / Package System** 

Sistem HARUS:

**7.5 QR Code Ticket (WAJIB)** 

Sistem HARUS:

**7.6 Check-in System (WAJIB)** 

Sistem HARUS:

• mencatat kehadiran peserta,  

• mencegah check-in ganda.  

**7.7 Lucky Draw (WAJIB)** 

Sistem HARUS: 

• memilih peserta secara acak,  

• menampilkan pemenang,  

• menggunakan data peserta yang valid.  

**7.8 Certificate Generator (WAJIB)** 

Sistem HARUS: 

• menghasilkan sertifikat digital,  

• menyimpan data sertifikat,  

• menampilkan sertifikat ke peserta.  

**7.9 Certificate Verification (WAJIB)** 

Sistem HARUS: 

• menyediakan QR atau ID unik,  

• memungkinkan verifikasi sertifikat,  

• memastikan sertifikat valid.
