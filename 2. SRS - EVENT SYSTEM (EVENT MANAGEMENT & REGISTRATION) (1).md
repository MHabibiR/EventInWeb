 **SYSTEM REQUIREMENT SPECIFICATION (SRS)** 

**FINAL PROJECT — EVENT SYSTEM (EVENT MANAGEMENT & REGISTRATION)** 

**1\. TUJUAN SISTEM** 

Kalian WAJIB membangun sistem Event Management yang: • dapat digunakan oleh peserta dan admin,  

• mampu mengelola event dari pendaftaran hingga selesai,  • mampu mengatur kapasitas dan seat/slot,  

• mampu menghasilkan QR ticket,  

• mampu melakukan check-in berbasis QR,  

• mampu menjalankan lucky draw,  

• mampu menghasilkan sertifikat digital,  

• mampu melakukan verifikasi sertifikat,  

• berjalan terintegrasi antara mobile app, web dashboard, dan backend.  

**2\. DEFINISI ROLE SISTEM** 

Kalian WAJIB mengimplementasikan 2 role: 

**2.1 Participant (Mobile)** 

Participant WAJIB bisa: 

• melihat event  

• mendaftar event  

• memilih kategori  

• memilih seat/slot (jika tersedia)  

• mendapatkan QR ticket  

• melakukan check-in   
• melihat status kehadiran  

• melihat sertifikat  

• memverifikasi sertifikat  

**2.2 Admin / Organizer (Web)** 

Admin WAJIB bisa: 

• login  

• membuat event  

• mengelola event  

• mengelola peserta  

• mengatur kategori  

• mengatur seat/slot  

• memonitor pendaftaran  

• memvalidasi check-in  

• menjalankan lucky draw  

• generate sertifikat  

• memonitor keseluruhan event  

**3\. MODUL SISTEM DAN REQUIREMENT DETAIL** 

**3.1 MODULE: AUTHENTICATION** 

**Requirement WAJIB:** 

• user dapat login  

• user dapat logout  

• sistem membedakan role:  

o participant   
o admin  

• user dapat update profile  

• user dapat mengganti password  **3.2 MODULE: EVENT MANAGEMENT** 

**3.2.1 Create Event** 

Admin WAJIB bisa: 

• membuat event  

• mengisi:  

o nama event  

o deskripsi  

o tanggal  

o lokasi  

o kapasitas peserta  

**3.2.2 Event Status** 

Event WAJIB memiliki status: 

• draft  

• open  

• closed  

• finished  

**3.2.3 Event Listing** 

Participant WAJIB bisa: 

• melihat daftar event   
• melihat detail event  

**3.3 MODULE: REGISTRATION** 

**3.3.1 Register Event** 

Participant WAJIB bisa: 

• mendaftar event  

• sistem harus:  

o mengecek kapasitas  

o menolak jika penuh  

**3.3.2 Participant Data** 

Sistem WAJIB menyimpan: 

• nama  

• email  

• kategori  

• seat (jika ada)  

**3.4 MODULE: CATEGORY MANAGEMENT** 

**Requirement WAJIB:** 

Sistem HARUS: 

• mendukung kategori peserta  

• memungkinkan admin membuat kategori  • memungkinkan participant memilih kategori   
**3.5 MODULE: SEAT / SLOT MANAGEMENT (WAJIB)** 

**Requirement WAJIB:** 

Sistem HARUS: 

• menampilkan seat atau slot  

• memastikan seat tidak dipilih lebih dari satu orang  • menampilkan status seat:  

o available  

o booked  

**3.6 MODULE: QR CODE TICKET (WAJIB)** 

**Requirement WAJIB:** 

Sistem HARUS: 

• menghasilkan QR code unik untuk setiap participant  • QR harus terhubung dengan data participant  • QR digunakan untuk check-in  

**3.7 MODULE: CHECK-IN SYSTEM (WAJIB)** 

**Requirement WAJIB:** 

Sistem HARUS: 

• membaca QR code  

• memvalidasi QR  

• mencatat waktu check-in  

• mencegah check-in lebih dari satu kali   
**Participant HARUS bisa:** 

• melihat status check-in  

**3.8 MODULE: LUCKY DRAW (WAJIB)** 

**Requirement WAJIB:** 

Sistem HARUS: 

• memilih peserta secara acak  

• hanya memilih peserta yang valid (terdaftar atau check-in jika ditentukan)  • menampilkan hasil pemenang  

**3.9 MODULE: CERTIFICATE GENERATION (WAJIB)** 

**Requirement WAJIB:** 

Sistem HARUS: 

• menghasilkan sertifikat digital  

• menyimpan data sertifikat  

• menampilkan sertifikat ke participant  

**Sertifikat HARUS berisi:** 

• nama peserta  

• nama event  

• tanggal event  

• ID atau QR verifikasi   
**3.10 MODULE: CERTIFICATE VERIFICATION (WAJIB)** 

**Requirement WAJIB:** 

Sistem HARUS: 

• menyediakan ID unik atau QR  

• memungkinkan validasi sertifikat  

• menampilkan status:  

o valid  

o tidak valid  

**3.11 MODULE: ADMIN DASHBOARD** 

**Admin WAJIB bisa melihat:** 

• jumlah event  

• jumlah peserta  

• kapasitas event  

• jumlah peserta terdaftar  

• jumlah peserta check-in  

• statistik event  

**3.12 MODULE: MONITORING** 

**Admin WAJIB bisa:** 

• melihat daftar peserta  

• melihat status peserta  

• melihat siapa yang sudah check-in   
• melihat siapa yang belum check-in  **4\. DATA REQUIREMENT**

**5\. SYSTEM FLOW** 

**5.1 Flow Registration** 

1\. participant melihat event  

2\. participant mendaftar  

3\. sistem cek kapasitas  

4\. data disimpan  

5\. QR ticket dibuat  

**5.2 Flow Seat Selection** 

1\. participant memilih seat  

2\. sistem cek ketersediaan  

3\. seat dikunci  

4\. data disimpan  

**5.3 Flow Check-in** 

1\. QR di-scan  

2\. sistem validasi  

3\. status check-in disimpan  

4\. participant dianggap hadir  

**5.4 Flow Lucky Draw** 

1\. admin menjalankan lucky draw  2\. sistem memilih peserta secara random   
3\. hasil ditampilkan  

**5.5 Flow Certificate** 

1\. event selesai  

2\. sistem generate sertifikat  

3\. participant melihat sertifikat  

**5.6 Flow Verification** 

1\. QR / ID dimasukkan  

2\. sistem mencari data  

3\. sistem menampilkan status valid  **6\. NON-FUNCTIONAL REQUIREMENT** 

**6.1 Performance** 

• response API \< 3 detik  

**6.2 Security** 

• password di-hash  

• validasi input wajib  

• QR tidak mudah ditebak  

**6.3 Usability** 

• UI jelas  

• seat mudah dipahami  

• QR mudah digunakan   
**6.4 Reliability** 

• sistem tidak crash saat demo  • check-in berjalan stabil

Project dianggap berhasil jika: • participant bisa daftar  

• kapasitas terkontrol  

• seat tidak bentrok  

• QR dibuat  

• check-in berhasil  

• lucky draw berjalan  

• sertifikat dihasilkan

• tidak memiliki QR,  

• tidak memiliki check-in,  

• tidak memiliki seat control,  

• tidak memiliki lucky draw,  

• tidak memiliki sertifikat,  

maka project dianggap **tidak memenuhi standar Event System**.
